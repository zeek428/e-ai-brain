# AI Brain 帮助中心

`docs/08-help/` 存放面向 AI Brain 最终用户的帮助文档、业务模块操作手册和配套截图。这里不是 PRD、技术规格或 API 契约；写作目标是让普通用户知道“这个功能做什么、我应该怎么操作、遇到问题怎么处理”。

系统内已提供“帮助中心”入口：登录后点击顶部“帮助中心”或头像菜单中的“帮助中心”，即可打开 `/help` 在线查看和搜索操作说明。

## 维护说明

## P0 维护规则

当开发迭代影响以下任一内容时，必须检查并同步帮助文档：

- 菜单入口、页面名称、页面布局或 Tab 分区变化。
- 按钮、字段、表格列、筛选条件、弹窗、抽屉或操作流程变化。
- 角色权限、只读/可编辑边界、审批/确认规则变化。
- 用户可见错误提示、错误码、空状态、成功提示变化。
- 页面截图或截图中的关键操作位置变化。

如果确认无需更新帮助文档，需要在最终交付说明中写明“帮助文档无需更新”及原因。

## 单篇手册模板

```md
# 模块名称

## 模块用途
说明该模块解决什么问题，适合哪些角色使用。

## 适用角色
列出可访问角色、只读角色和可写角色。

## 页面总览
插入页面截图，并用编号说明核心区域。

## 核心概念
解释状态、字段、指标和关键业务对象。

## 常用操作
1. 操作入口
2. 填写字段
3. 提交或确认
4. 查看结果、全链路或审计记录

## 注意事项
说明权限边界、不可逆动作、数据归属和常见误区。

## 常见问题
列出常见错误、原因和处理方式。

## 相关功能
链接到上下游模块。
```

## 截图规范

截图放在 `docs/08-help/assets/screenshots/`，文件名使用页面路径和用途命名：

```text
assets-knowledge-overview.png
assets-knowledge-upload-form.png
system-settings-email-test.png
delivery-requirements-approve.png
```

截图要求：

- 使用真实本地 Web 页面截图，优先选择当前稳定开发环境。
- 每个核心模块至少保留一张页面总览截图。
- 核心操作流程可以配 2-5 张步骤截图。
- 截图中的邮箱、手机号、真实姓名、token、密码、密钥、内外网域名、Cookie、连接串等必须脱敏。
- 截图只标注用户需要关注的按钮、字段、状态和结果，不标注内部实现细节。
- 页面改版后，必须检查旧截图是否仍然准确。

## 开发完成检查清单

每次用户可见功能迭代完成后，执行以下检查：

- [ ] 是否新增或修改菜单入口？
- [ ] 是否新增或修改页面字段、按钮、表格列、筛选项？
- [ ] 是否改变核心操作流程或审批确认流程？
- [ ] 是否改变角色权限、只读/编辑边界？
- [ ] 是否新增或修改错误码、错误提示、空状态？
- [ ] 是否需要更新页面总览截图或操作步骤截图？
- [ ] 是否需要更新 FAQ？
- [ ] 是否已在 `docs/changelog.md` 记录帮助文档同步情况？

## 与知识中心的关系

`docs/08-help/` 是帮助文档的源码目录。后续可以把这些文档同步到 AI Brain 知识中心的“系统帮助”空间，让 AI 助手基于帮助文档回答“这个功能怎么用”。在同步机制落地前，以本目录中的 Markdown 和截图为准。
