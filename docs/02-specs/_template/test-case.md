## 测试用例规范

### 用例编号规则
```
TC-{模块}-{功能}-{类型}-{序号}

类型:
- FUNC: 功能测试
- BOUND: 边界测试
- ERR: 异常测试
- API: 接口测试
- PERF: 性能测试
```

### 用例优先级
| 优先级 | 说明 | 通过要求 |
|--------|------|----------|
| P0 | 核心功能，阻塞发布 | 必须 100% 通过 |
| P1 | 重要功能，影响主流程 | 必须 100% 通过 |
| P2 | 一般功能，影响体验 | ≥ 95% 通过 |
| P3 | 边缘场景 | ≥ 90% 通过 |
