## 如何贡献

### 报告问题
1. 先检查当前任务和文档是否已有相同问题
2. 使用 Issue 模板或项目约定渠道提交
3. 提供复现步骤和环境信息

### 提交代码

#### 分支命名
```
feature/{功能名称}    # 新功能
fix/{问题描述}        # Bug 修复
refactor/{重构内容}   # 重构
docs/{文档内容}       # 文档更新
```

#### 提交信息格式
```
<type>: <description>

[optional body]

type: feat | fix | refactor | docs | test | chore | perf
```

#### 开发流程
1. Fork 或创建分支
2. 编写代码 + 测试
3. 提交 PR
4. 等待 Review
5. 合并到主分支

### 代码规范
详见 [编码规范](../06-standards/coding-standards.md)

### 测试要求
- 新功能必须有测试
- 测试覆盖率 ≥ 80%
- 所有测试必须通过
- P0/P1 用例必须 100% 通过

### 测试用例要求
- 按 [测试规范](../06-standards/testing.md) 编写用例
- 用例编号格式: `TC-{模块}-{功能}-{类型}-{序号}`
- 每个验收标准至少对应一个测试用例
- 详见 [测试用例模板](../02-specs/_template/test-case.md)

## Review 标准
- 代码质量
- 测试覆盖
- 测试用例完整
- 文档更新
- 无安全漏洞
