# AI 自动开发标准流程图

本文档通过流程图展示 AI 辅助开发的完整流程，帮助理解各阶段的工作内容和文档流转。

## 流程总览

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        AI 自动开发标准流程                                    │
└─────────────────────────────────────────────────────────────────────────────┘

  ┌─────────┐     ┌─────────┐     ┌─────────┐     ┌─────────┐
  │ 需求分析 │────▶│ 技术设计 │────▶│ 代码实现 │────▶│ 验收交付 │
  └─────────┘     └─────────┘     └─────────┘     └─────────┘
       │               │               │               │
       ▼               ▼               ▼               ▼
   PRD 文档        Spec 文档        源代码          Changelog
   + 高保原型      (技术方案)       (实现)          (变更记录)
   (需求定义)
```

## 快速参考

### 各阶段关键输出物

| 阶段 | 主要文档 | 状态要求 |
|------|----------|----------|
| 需求分析 | `01-prd/{feature}/prd.md` | Approved |
| 需求分析 | `01-prd/{feature}/prototype/index.html` | Approved |
| 技术设计 | `02-specs/{feature}/spec.md` | Approved |
| 技术设计 | `02-specs/{feature}/api.md` | Approved |
| 技术设计 | `02-specs/{feature}/test-case.md` | Approved |
| 代码实现 | `src/*`, `tests/*` | 测试通过 |
| 验收交付 | `02-specs/{feature}/test-report.md` | 人工复核通过 |
| 验收交付 | `changelog.md` | 已更新 |

### 关键检查点

```
□ PRD 阶段
  ├── 用户故事完整
  ├── 验收标准明确
  ├── In/Out Scope 清晰
  └── 高保原型已确认

□ Spec 阶段
  ├── 设计决策有依据
  ├── 数据库设计完整
  ├── API 定义完整
  └── 测试策略明确

□ 实现阶段
  ├── 代码符合设计
  ├── 单元测试覆盖率 ≥ 80%
  └── 代码规范检查通过

□ 验收阶段
  ├── AI 测试执行完成
  ├── 测试报告已生成
  ├── 人工复核测试报告通过
  └── 人工验收测试通过

□ 交付阶段
  ├── Code Review 通过
  └── 文档已同步更新
```
